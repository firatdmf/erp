{% trans "⚡ Quick Start (in 5 minutes)" %}

{% blocktrans %}The system opens one Current Account card for each customer and supplier (example code: CARI-001). Under this card:{% endblocktrans %}

  • {% blocktrans %}Movements — each financial effect is a row (invoice, collection, endorsement, etc.){% endblocktrans %}
  • {% blocktrans %}Balance — the sum of movements, updated automatically{% endblocktrans %}
  • {% blocktrans %}Invoices — all sales/purchase invoices issued for this account{% endblocktrans %}
  • {% blocktrans %}Collections / Payments — transactions where money changed hands{% endblocktrans %}
  • {% blocktrans %}Checks / Promissory Notes — negotiable instruments waiting in the portfolio{% endblocktrans %}

{% trans "First 3 steps" %}

  1. {% url 'current_account:cari_create' as cari_create_url %}{% blocktrans %}Open an account card: New Account → select customer or supplier, enter the previous opening balance if any.{% endblocktrans %}
  2. {% blocktrans %}Issue an invoice: On the account page, "Issue Invoice" purple button → add line items → "Save & Issue".{% endblocktrans %}
  3. {% blocktrans %}Take a collection: On the account page, "Take Collection" green button → open invoices are listed automatically, match them.{% endblocktrans %}

{% trans "That's all. Balance, statement, and status update automatically." %}

{% trans "📋 Daily Scenarios" %}

{% trans "A — I make a sale to a customer, then I receive the money" %}

  1. {% blocktrans %}Open an account (skip if it already exists): /cari/yeni/{% endblocktrans %}
  2. {% blocktrans %}Issue a sales invoice: On the account detail page, "Issue Invoice" → Type: Sales Invoice{% endblocktrans %}
    • {% trans "Add line items (each row has a VAT rate)" %}
    • {% trans 'Click "Save & Issue" → the account balance increases automatically' %}
  3. {% blocktrans %}When the customer pays: Account → "Take Collection"{% endblocktrans %}
    • {% trans "Open invoices are listed automatically" %}
    • {% trans '"Auto Match (FIFO)" closes the oldest invoice(s)' %}
    • {% trans 'Or match manually → "Save & Confirm"' %}
  4. {% blocktrans %}Result: Invoice status becomes Paid, the account balance closes, the cash account increases.{% endblocktrans %}

{% trans "B — I bought goods from a supplier, I will make a payment" %}

  1. {% blocktrans %}Open an account: Type = Supplier{% endblocktrans %}
  2. {% blocktrans %}Purchase invoice: Type: Purchase Invoice → the account balance becomes negative{% endblocktrans %}
  3. {% blocktrans %}Payment: Account → "Take Collection" → Type: Payment (to supplier){% endblocktrans %}
    • {% trans "Select the account to deduct from the cash account, close the invoice" %}
  4. {% blocktrans %}Result: The account balance returns to zero, the cash account decreases.{% endblocktrans %}

{% trans "C — The customer brought a check" %}

  1. {% blocktrans %}Check entry: /cari/cek-senet/yeni/ → Direction: Received{% endblocktrans %}
    • {% trans 'Enter bank, serial number, due date, amount → "Add to Portfolio"' %}
  2. {% blocktrans %}Automatic effect: The account balance decreases (as if you had taken a collection).{% endblocktrans %}
  3. {% blocktrans %}When the due date arrives, 3 options:{% endblocktrans %}
    • {% blocktrans %}Deposit → status: Deposited{% endblocktrans %}
    • {% blocktrans %}Clear → select cash account → money comes in{% endblocktrans %}
    • {% blocktrans %}Endorse → transfer to another account{% endblocktrans %}
  4. {% blocktrans %}If it bounces: "Mark as Bounced" → a reverse movement is written to the account, and the receivable is reopened.{% endblocktrans %}

{% trans "🧠 Core Concepts" %}

{% trans "Sign convention" %}

{% trans "The heart of the system is built on a single rule:" %}

amount{% trans "Meaning" %}
{% trans "Positive (+)" %}{% blocktrans %}The account owes us / we are the creditor{% endblocktrans %}
{% trans "Negative (−)" %}{% blocktrans %}We owe the account / the account paid us{% endblocktrans %}
0{% trans "Account is closed" %}

{% trans "Movement → Balance" %}

{% trans "Balance is not a stored field, it is the sum of movements:" %}

Balance = SUM(CariMovement.amount) WHERE cari = X

{% blocktrans %}That's why when you delete / change a movement, the balance auto-corrects. The cancel action does not delete the original row, it writes a reverse movement (audit-safe).{% endblocktrans %}

{% trans "Status flows" %}

{% trans "Invoice:" %}

draft → issued → partially paid → paid
                 └─→ cancelled

{% trans "Check/Promissory Note (received):" %}

in portfolio ──┬─► deposited ──► cleared
               ├─► cleared
               ├─► endorsed
               └─► bounced / cancelled

  • {% blocktrans %}CARI-001, CARI-002, ... are assigned automatically{% endblocktrans %}
  • {% trans "Two different books can have the same code — unique as a (book, code) combination" %}

{% trans "Issuing flow" %}

  1. {% blocktrans %}Create a draft: You add line items, VAT is calculated per row{% endblocktrans %}
  2. {% blocktrans %}"Issue": Status becomes issued, CariMovement is created automatically{% endblocktrans %}
  3. {% blocktrans %}"Cancel" (if issued): A reverse movement is written, the original is not deleted{% endblocktrans %}
{% trans "A draft invoice does not affect the current account. It only takes effect the moment you issue it." %}

{% trans "VAT and discount" %}

{% trans "Defined separately on each line:" %}

  • {% blocktrans %}Discount %: applied first (deducted from the line total){% endblocktrans %}
  • {% blocktrans %}VAT %: added on top of the net after discount{% endblocktrans %}
  • {% trans "Different VAT rates can exist on the same invoice (e.g., some items at 20%, some at 10%)" %}
  • {% blocktrans %}The VAT breakdown is automatically shown on the detail page{% endblocktrans %}
{% blocktrans %}Only draft invoices can be edited. After issuing, the invoice "freezes" — to edit, you must cancel and re-issue it.{% endblocktrans %}

{% trans "❓ Frequently Asked Questions" %}

{% trans "I issued the wrong invoice, what do I do?" %}

{% blocktrans %}If it's a draft: "Delete" button — cleanly deleted.
If it's issued: "Cancel" — a reverse account movement is written automatically, the original record remains (audit-safe). Then issue the correct one as a new invoice.{% endblocktrans %}

{% trans "I entered a collection wrong?" %}

{% trans "Same logic: if not confirmed, delete; if confirmed, cancel. Cancel reverses everything (account, invoice status, cash account)." %}

{% trans "I use my customer as a supplier too — should I open two separate accounts?" %}

{% blocktrans %}No. Mark the account type as "Customer & Supplier". All movements are collected on a single card, the balance is automatically net.{% endblocktrans %}

{% trans "Can I change the opening balance later?" %}

{% blocktrans %}Not directly, but the equivalent: on the account card, use "Add Movement" to enter a positive/negative adjustment movement, and write "Opening adjustment" in the description.{% endblocktrans %}

{% trans "What happened to the Receivables/Payables in my old accounting module?" %}

{% blocktrans %}All of them were automatically migrated to Account cards (Phase 1 migration). The old tables still exist and new account movements are automatically written to them as well. We didn't break the old dashboards.{% endblocktrans %}

{% trans "Can I get the statement as a PDF?" %}

{% blocktrans %}Use your browser's print function (Ctrl+P) → "Save as PDF". The page is optimized with @media print.{% endblocktrans %}

{% trans "Can I split a collection across multiple invoices?" %}

{% trans 'Yes. In the matching table, you write whatever amount you want to each invoice, or "Auto Match (FIFO)" closes the oldest invoices in sequence.' %}

{% trans "Can a portion of the collection remain as an advance?" %}

{% blocktrans %}Yes. If the collection amount is greater than the sum of matches, the difference is automatically recorded as an advance.{% endblocktrans %}

{% trans "I deposited the check, when does it hit the cash account?" %}

{% blocktrans %}Depositing is just a status change. When the money actually arrives, you select the cash account with "Clear", and only then does the cash account balance increase.{% endblocktrans %}

{% trans "Multi-tenant — does this module support it?" %}

{% blocktrans %}It does. Each Book behaves like a separate business/brand. When ENV_PROFILE changes and the schema changes, all accounts/movements remain isolated.{% endblocktrans %}

{% blocktrans %}This guide is continuously updated. The moment you see something missing, report it or check the Current Account module changes.{% endblocktrans %}
